
Connect to EC2 instance
use command 'lsblk' to see all the attached drives
use command 'sudo file -s /dev/xvdb' to see special information
	If output is 'data', there is no file system on drive.

use command 'sudo mkfs -t ext4 /dev/xvdb' to create ext4 file system on drive
use command 'sudo mkdir /data' to create a folder 'data'
use command 'sudo mount /dev/xvdb /data' to mount the drive to /data
use command 'lsblk' to check EBS volume status

use command 'sudo cp /etc/fstab /etc/fstab.orig' to mount EBS volume on every system reboot
use command 'sudo nano /etc/fstab' to go inside file to add information
'/dev/xvdb /data ext4 defaults,nofail 0 2' add this information inside fstab file
[Now after reboot, the EBS volume will automatically mount]
use command 'cat /etc/fstab' to check
use command 'sudo file -s /dev/xvdb' to verify the file system has been formatted

use command 'sudo umount /data' to unmount /data file [use cd ../ to come out of /data file first]
use command 'lsblk' to check EBS volume status

use command 'sudo mount -a' to unmount
use command 'lsblk' to check EBS volume status
